# Система управления проектами и брифами

## Концепция

Центральная система управления проектами позволяет **один раз** создать бриф о компании/продукте, после чего все модули автоматически используют эту информацию.

## Основные преимущества

1. **Один бриф для всех модулей** - не нужно вводить информацию о бизнесе в каждом модуле
2. **Передача данных между модулями** - результаты одного модуля используются в других
3. **История работы** - все результаты сохраняются в контексте проекта
4. **Быстрый старт** - загрузили бриф → запустили все модули

## Структура брифа

### Основная информация
- **Название проекта** - для идентификации
- **Название бизнеса** - юридическое/торговое название
- **Ниша бизнеса** - сфера деятельности
- **Описание бизнеса** - чем занимаетесь, что предлагаете

### Целевая аудитория
- **Демография** - возраст, пол, география
- **Интересы** - что интересует ЦА
- **Боли и проблемы** - какие проблемы решаете
- **Желания и цели** - к чему стремится ЦА

### УТП и преимущества
- **Уникальное торговое предложение (УТП)** - главное отличие от конкурентов
- **Конкурентные преимущества** - список преимуществ

### Бюджет и цели
- **Бюджет** - сумма и период (день/неделя/месяц)
- **Цели кампании** - что хотите достичь
- **Целевая цена конверсии** - допустимая стоимость заявки/продажи

### Дополнительная информация
- **География показов** - где показывать рекламу
- **Расписание показов** - когда показывать
- **Запреты и ограничения** - что нельзя использовать

## Рабочий процесс

### 1. Создание проекта
1. Перейти в раздел "Проекты"
2. Нажать "Новый проект"
3. Заполнить бриф
4. Сохранить

### 2. Работа с модулями
Все модули автоматически используют данные из брифа:

#### Семантика
- Генерирует ключевые слова на основе ниши и описания бизнеса
- Результат сохраняется в проект

#### Креативы
- Использует информацию о ЦА и УТП
- Генерирует креативные идеи для РСЯ
- Учитывает боли и желания аудитории

#### Объявления
- **Использует результаты семантики** (ключевые слова)
- Создает заголовки и тексты на основе УТП
- Учитывает запреты и ограничения

#### Минус-слова
- Анализирует на основе ниши и описания бизнеса
- **Может использовать семантику** для генерации минус-слов
- Сохраняет результаты анализа

#### Кампании
- **Использует результаты всех модулей**:
  - Семантику для структуры групп
  - Объявления для текстов
  - Минус-слова для фильтрации
- Создает структуру согласно базе знаний
- Учитывает бюджет и географию

#### Стратегия
- Разрабатывает план на основе бюджета и целей
- Использует информацию о ЦА
- Предлагает распределение бюджета

### 3. Последовательность работы (рекомендуется)

```
1. Создать проект и заполнить бриф
   ↓
2. Сгенерировать семантику
   ↓
3. Сгенерировать креативы (для РСЯ)
   ↓
4. Создать объявления (используют семантику)
   ↓
5. Сделать анализ минус-слов
   ↓
6. Создать структуру кампаний (использует всё)
   ↓
7. Разработать стратегию запуска
```

## API Endpoints

### Управление проектами

```
POST /api/projects/create
- Создание нового проекта
- Body: { name, brief }

GET /api/projects/list
- Список всех проектов пользователя

GET /api/projects/:projectId
- Получение конкретного проекта

PUT /api/projects/:projectId
- Обновление проекта/брифа

DELETE /api/projects/:projectId
- Удаление проекта

GET /api/projects/:projectId/:module
- Получение результатов модуля
- module: semantics | creatives | ads | minusWords | campaigns | strategy
```

## Хранение данных

В каждом проекте сохраняются:

```typescript
{
  id: string;
  userId: string;
  name: string;
  brief: ProjectBrief;

  // Результаты модулей
  semantics?: {
    keywords: string[];
    generatedAt: Date;
  };
  creatives?: {
    ideas: any[];
    generatedAt: Date;
  };
  ads?: {
    headlines: string[];
    texts: string[];
    generatedAt: Date;
  };
  minusWords?: {
    words: string[];
    analysis?: any;
    generatedAt: Date;
  };
  campaigns?: {
    structure: any;
    generatedAt: Date;
  };
  strategy?: {
    plan: any;
    generatedAt: Date;
  };
}
```

## Передача данных между модулями

### Пример: От семантики к объявлениям

1. Пользователь генерирует семантику
2. Результат сохраняется в `project.semantics.keywords`
3. При открытии модуля "Объявления":
   - Автоматически загружаются ключевые слова из проекта
   - Используются для генерации заголовков и текстов
   - Не нужно вводить заново

### Пример: Полный цикл кампании

1. **Бриф** → содержит всю информацию о бизнесе
2. **Семантика** → генерирует ключевые слова по брифу
3. **Объявления** → создает тексты по ключам + УТП из брифа
4. **Минус-слова** → анализирует по ключам + ниша из брифа
5. **Кампании** → собирает всё вместе:
   - Ключи из семантики
   - Тексты из объявлений
   - Минус-слова для фильтрации
   - Бюджет и гео из брифа
